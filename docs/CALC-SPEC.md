# CALC-SPEC.md — расчетное ядро (ПП №354, пени, перерасчеты)

Источник: дизайн Plan-агента + юр-верификация 8-агентного ресёрча, 2026-07-18. Ядро — чистый пакет `packages/billing-core`: вход — frozen-контекст со ВСЕМИ данными, выход — строки с трейсом. Времени «сейчас» в ядре нет (запрещено линтером). Деньги — Decimal (decimal.js), float запрещен.

Пометка **[проверить]** = сверить пункт/номер формулы с действующей редакцией ПП 354 (КонсультантПлюс) непосредственно перед кодированием формулы; результат сверки — в докстринг (пункт + дата).

## 1. Каталог расчетных случаев

### 1.1 Отопление

Обозначения: `Si` — площадь помещения; `Sоб` — сумма площадей всех помещений; `NT` — норматив (Гкал/м²·мес); `TT` — тариф (руб/Гкал); `Vд` — объем по ОДПУ; `Vi` — по ИПУ; `K` — коэффициент периодичности = месяцы отопительного периода / 12.

| ID | Ситуация | Формула прил. 2 ПП 354 | Примечания |
|---|---|---|---|
| H-01 | Нет ОДПУ; оплата в отопительный период | **ф.2** (ред. после ПП 184-2019 дополнена членами за общее имущество и вычетом инд. источников **[проверить точную запись]**) | вне сезона начисление = 0 |
| H-02 | Нет ОДПУ; равномерно за год | **ф.2(1)**: `Si × (NT × K) × TT` | различие ф.2 и 2(1) — СПОСОБ ОПЛАТЫ (не тип здания!); K — только из НПА региона |
| H-03 | Нет ОДПУ; регион сохранил нормативы «до-2012» | ф.2(3)/2(4) **[проверить какая какому способу]** | конфигурационный флаг территории |
| H-04 | ОДПУ есть; ИПУ ни в одном помещении; сезон | **ф.3**: `Vд × Si/Sоб × TT` | Σ по дому == Vд — балансировка (метод наибольших остатков) |
| H-05 | То же; равномерно | ф.3 с Vд = среднемесячный за прошлый год | обязательна корректировка H-08 |
| H-06 | ОДПУ + ИПУ в части помещений (после ПКС №30-П от 10.07.2018 + ПП 1708) | **ф.3(1)**; замещающие объемы для помещений без ИПУ — вспомогательная формула **[проверить: 3(6)/3(7)]** | небаланс `(Vд − ΣVj) < 0` → принять 0, зафиксировать в трейсе |
| H-07 | ОДПУ + ИПУ во ВСЕХ помещениях | **ф.3(3)** | не путать: 3(3) — «все с ИПУ», а не способ оплаты |
| H-08 | Годовая корректировка при оплате 1/12 | **ф.3(4)** (ф.3(2) УТРАТИЛА силу с 01.01.2019, ПП 1708) — I квартал следующего года | может быть отрицательной; при смене собственника внутри года — корректировка владевшему, пропорционально |
| H-09 | Помещение с законным индивидуальным источником тепла (после ПКС №46-П) | действующие ред. ф.2(3)–2(6), 3, 3(1) с Sинд **[проверить номера]** | личное потребление 0, платится доля на общее имущество; случай массовый |
| H-10 | Повышающий коэффициент к отоплению | **НЕ применяется** — отменен ПП №232 от 27.02.2017 | правило версионируется (для перерасчетов 2016 существовал) |
| H-11 | ОДПУ вышел из строя / истек МПИ | п. 59(2): среднемесячный (окно ≥6 мес **[проверить]**), ≤3 периодов → норматив | состояние ОДПУ — конечный автомат с датами |
| H-12 | Нежилое помещение в МКД | те же формулы (п. 42(1), 43); с 2017 — прямые договоры с РСО | |
| H-13 | Регион сменил способ оплаты с 1 января | переходная корректировка + смена K | редкий, но ломающий кейс — покрыть тестом |

### 1.2 ГВС

Двухкомпонентность — свойство ТАРИФА, ортогональное способу определения объема: сначала объем `Vi` (W-01…W-07), потом деньги (D-1/D-2).

| ID | Ситуация | Правило | Примечания |
|---|---|---|---|
| W-01 | ИПУ исправен, показания в срок | объем = разность показаний | переход через 0 по значности счетчика |
| W-02 | ИПУ нет, техвозможность есть | **ф.4 + п.42**: `ni × Nгв × Кпов(1,5) × T` | Кпов — отдельная строка/атрибут UPLIFT, не «раздутый норматив» (субсидии/отчетность) |
| W-03 | ИПУ нет, техвозможности нет (акт) | ф.4 без Кпов | критерии — приказ №627 от 29.12.2011 **[проверить актуальный]** |
| W-04 | Показания не переданы | п.59(б): среднемесячный за **≥6 мес** (если ПУ работал меньше — факт, но ≥3 мес), **≤3 периодов подряд** → п.60: норматив **БЕЗ Кпов** | классические ошибки: окно 3 вместо 6; несброшенный счетчик периодов |
| W-05 | ИПУ вышел из строя / истек МПИ | п.59(а): среднее **с даты события** (месяц делится), ≤3 периодов → п.60: норматив **× Кпов 1,5** | отличие от W-04 — Кпов есть |
| W-06 | Недопуск исполнителя к проверке ПУ | акт недопуска → через 3 мес норматив × Кпов **[проверить пункт 60/85(3)]** | |
| W-07 | Зарегистрированных нет (ni=0) | **п.56(2)**: по числу СОБСТВЕННИКОВ | проверять всегда при ni=0 без ИПУ |
| D-1 | Однокомпонентный тариф | `P = V × Tгвс` | |
| D-2 | Двухкомпонентный (закрытая система) | **ф.23**: `P = V×Tхв + (V×qкр)×Tтэ` | **qкр — региональный НОРМАТИВ подогрева (~0,05–0,07 Гкал/м³), НЕ фактический удельный расход** (позиция ВС РФ — ловушка №1); Кпов — только к компоненту холодной воды |
| D-2а | Открытая система | компонент «теплоноситель» вместо ХВ | **[проверить распределение ф.24/26]** |
| W-08 | ОДН-ГВС (РСО — исполнитель КУ) | ф.24 (двухкомп. ОДН) | этап 2; уточнить наличие таких домов у пилота |
| W-09 | Временное отсутствие | раздел VIII, п.86–97: >5 ПОЛНЫХ дней подряд; ТОЛЬКО без ИПУ при акте невозможности установки; заявление до отъезда или ≤30 дней после; авансом ≤6 мес; исполнение 5 раб. дней; НЕ отопление никогда | результат — отрицательная строка RECALC |
| W-10 | Контрольные/поздние показания | **п.61**: разница — доначисление/снятие в периоде обнаружения | механика — §3 |
| W-11 | Нежилое в МКД без ПУ (вода) | расчетные способы ПП 776 | этап 2, каркас |

### 1.5 Пени физлиц (ст. 155 ч.14 ЖК РФ)

Шкала от срока оплаты (**с 01.03.2026 по 177-ФЗ срок — до 15-го числа**; до этого — 10-е; версионируемый параметр):

| Дни просрочки | Доля ключевой ставки |
|---|---|
| 1–30 | 0 |
| 31–90 | 1/300 в день |
| 91+ | 1/130 в день |

День фактической оплаты включается. Границы 31/91 — от срока оплаты КАЖДОГО месячного требования отдельно.

**Ставка**: min(ключевая на день фактической оплаты; **9,5%**) — льготный порядок с 28.02.2022, продлен на 2025–2026 **ПП №329 от 18.03.2025**, действует до 01.01.2027. НЕ хардкодить: `PenaltyRatePolicy` — версионируемые данные (ставка, cap, мораторные окна: 06.04–31.12.2020 ПП 424; 01.04–01.10.2022 ПП 497).

**Механика (главное место ошибок самодельных биллингов):**
1. Долг ведется по **требованиям** (ЛС, услуга, месяц, срок оплаты, сумма); перерасчеты меняют сумму требования с даты документа.
2. Оплата привязывается: по назначению платежа; без назначения — к периодам с неистекшей исковой давностью начиная с раннего (п.32 Пленума ВС №22, ст.319.1 ГК).
3. По требованию строится **таймлайн событий**: срок оплаты; дни 31 и 91; каждая частичная оплата (уменьшает остаток со дня, СЛЕДУЮЩЕГО за днем оплаты); смены ставки/cap; мораторные границы; изменения суммы.
4. Таймлайн режется на интервалы постоянства → пеня = Σ `остаток × ставка × доля × дни`. Округление до копейки один раз — на итог по требованию за период.
5. Изменение начисления задним числом → полный пересчет пени требования, дельта — документом текущего периода.

Никаких «пеня = долг × дни × ставка» одним умножением.

## 2. Архитектура ядра

```
packages/billing-core/src/
  model/    context.ts (frozen-контекст), result.ts (AccrualLine + TraceStep), enums.ts
  rules/    ruleset.ts (реестр редакций методики по датам), selector.ts (выбор метода + обоснование)
  methods/  heating.ts, hot-water.ts, contract.ts
  penalty/  engine.ts (интервальный движок), policy.ts (ставки/cap/моратории/шкалы)
  recalc/   replay.ts (replay + diff → дельта)
  rounding.ts  splitter.ts (DateSplitter)  gost-qr.ts [Ф0 ✅]
```

- **Селектор**: один декларативный список правил (предикаты → MethodId) с машиночитаемым обоснованием («ОДПУ: да; ИПУ: 12 из 40; способ: равномерно → ф.3(1)») — в трейс и в ответ абоненту. Тестируется полной таблицей решений.
- **Версионирование методики**: изменение закона = НОВЫЙ класс формулы с датой начала действия; старые реализации не редактируются. Расчет за январь-2025, запущенный в 2027, дает бит-в-бит тот же результат (golden с замороженным временем). Данные (Кпов, cap, срок оплаты, K) — версионируемые интервалы, выбор всегда `as_of = расчетный период`.
- **Контекст** (эскиз): period, account (тип, Si, доли), house (Sоб, Sои, ОДПУ и агрегаты ИПУ, сезон, способ оплаты, K, инд. источники), occupancy-интервалы + собственники, meters (конечный автомат состояний: OK/NO_READINGS(n)/BROKEN/EXPIRED/NO_ACCESS/ABSENT+акт, история ≥6 мес), tariffs/normatives (интервальные, компоненты, qкр), absences, prior_lines (для replay).
- **Результат**: строки {service, method, ruleset_version, sub_interval, volume, rate, amount, kind: ACCRUAL|RECALC|PENALTY|UPLIFT|VAT, trace, basis_refs}.

| Величина | Точность | Правило |
|---|---|---|
| Объемы воды | 3 знака (м³) | как переданы/вычислены |
| Объемы тепла | 6 знаков (Гкал) | |

## 3. Перерасчеты: replay + diff

Закрытые периоды immutable. (1) **Replay**: контекст месяца «как должно было быть» → расчет ПО ПРАВИЛАМ ТОГО месяца. (2) **Diff** с Σ ранее выпущенных строк месяца. (3) **Дельта** (±) — строкой RECALC текущего периода со ссылками и основанием. (4) Идемпотентность: повторный replay → дельта 0 (property-тест).

Частные случаи: поздние показания (интервал от последнего факта пересчитывается; объем-разница по месяцам пропорционально дням, каждый месяц по своим тарифам — политика с настройкой, упрощенный вариант «всё в текущий» допустим по п.61 **[сверить предпочтение с ГЖИ региона]**); жильцы задним числом (replay с новым ni); тариф задним числом (replay + каскад пересчета Кпов-строк и ПЕНЕЙ от новой базы).

## 4. Тестовая стратегия

**Golden-набор ~120 кейсов** (YAML: context, expected_lines до копейки, точки трейса, источник эталона, дата): отопление ≈25–30, ГВС ≈35–40, сплиты дат ≈10–15, пени ≈20–25 (границы 31/91, частичные оплаты, смена ставки, cap, мораторий, оплата в граничный день, перерасчет базы), юрлица ≈8–10, replay-цепочки ≈15–20. Эталоны по убыванию авторитетности: (1) калькулятор ЖКХ ГИС ЖКХ (dom.gosuslugi.ru) / ФАС — ≥30 кейсов; (2) числовые примеры писем Минстроя и разборов ГЖИ; (3) АКАТО/РосКвартал перекрестно; (4) двойной независимый ручной расчет.

**Property-based (fast-check)**: Σ объемов дома == ОДПУ; неотрицательность всех строк кроме RECALC; непрерывность (расчет разрезанного месяца == неразрезанному ± 1 коп × подстроки); пени: монотонность, частичная оплата уменьшает, мораторий = 0, независимость от порядка событий; replay-идемпотентность; независимость от «сегодня».

**Мутационное тестирование** (Stryker) для methods/ и penalty/; 100% branch coverage ядра — CI-гейт; табличный тест селектора (полная матрица предикатов).

**Параллельный прогон с пилотом**: 2–3 месяца, все ЛС, построчная сверка; каждое расхождение классифицируется (наша ошибка → фикс+регресс; ошибка старой системы → документируем; округление ≤1 коп). Отдельно месяц годовой корректировки и месяц смены тарифа (июль).

## 5. Топ-7 ловушек и защита

1. **Float/«копейка гуляет»** → Decimal везде, тип Money, одна точка округления, округление в трейсе.
2. **Пени** от полной суммы вместо остатка по дням; игнор частичных оплат; текущая ставка на все дни; забытый cap/мораторий → интервальный event-движок, версионируемая политика, golden на каждую границу.
3. **Кпов не там** (при непередаче показаний; к отоплению после 2017; вшит в норматив) → отдельная строка UPLIFT только в W-02/W-05/W-06; правило версионируется.
4. **Средний объем**: окно 3 вместо ≥6; несброшенный счетчик «3 периодов»; месяц события не делится → конечный автомат ИПУ с датами и счетчиком, табличный тест переходов.
5. **Баланс дома**: Σ ≠ ОДПУ из-за независимого округления; отрицательный небаланс в минус → наибольшие остатки; «небаланс < 0 → 0» с трейсом.
6. **Нет версионности**: история пересчитывается новыми данными → интервалы везде, as_of = период, редакции формул — классы с датами.
7. **Пропорция дней**: деление на 30; день события дважды/ни разу; K пересчитан по факту сезона → единый DateSplitter на полуинтервалах, фактические дни, K только из НПА.
7½. **ГВС-подогрев по факту** вместо норматива qкр → в ф.23 компонент ТЭ жестко `V × qкр`; фактического удельного расхода в контексте физлиц нет как понятия.
