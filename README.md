# Теплобиллинг

Система начислений ЖКХ (отопление + горячая вода) для МУП и РСО. Расчеты по Постановлению Правительства РФ №354, платежи и сальдо, пени, квитанции с QR по ГОСТ Р 56042-2014, выгрузки в ГИС ЖКХ и 1С.

**Живая версия:** https://a1exxx.github.io/teplobilling/

## Как это устроено (v1)

Первая версия работает **целиком в браузере**: внутри крутится настоящий PostgreSQL (PGlite/WASM), данные хранятся в IndexedDB вашего браузера и не покидают компьютер. Сервер не нужен — открыл ссылку и работаешь. Серверная версия (VPS + PostgreSQL) переиспользует тот же код: одна схема БД (Drizzle) и одно расчетное ядро (TypeScript).

### Ограничения v1 (честно)

- Однопользовательский режим: база живет в браузере конкретного компьютера. Перенос — через экспорт/импорт файла БД (появится в Ф6).
- Роли и аудит — условные до серверной версии (v2).
- Массовая печать 10 000 PDF и прямая интеграция с ГИС ЖКХ (SOAP + ЭЦП) — в v2; в v1 ГИС ЖКХ закрывается официальными Excel-шаблонами.

## Запуск локально

```bash
pnpm install
pnpm dev        # dev-сервер на vite
pnpm test       # юнит-тесты (vitest): расчетное ядро + миграции на PGlite
pnpm build      # прод-сборка
pnpm typecheck  # строгий TypeScript по всем пакетам
```

## Структура

```
packages/billing-core   # чистое расчетное ядро: формулы ПП №354, пени, QR ГОСТ (decimal.js, без I/O)
packages/db             # схема БД (Drizzle) + миграции: одна на PGlite (браузер) и PostgreSQL (сервер)
apps/web                # операторское приложение: React 19 + Ant Design v6, PGlite поверх IndexedDB
docs/                   # спецификации: план, архитектура, расчетные случаи, ресёрч, roadmap
tests/e2e               # Playwright-сценарии (SCENARIOS.md — контракт E2E)
```
